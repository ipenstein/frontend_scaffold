Best Practice Forms

Based on Form Kitchen Sink

This example meets the following requirements:

  • Container independent, flexible widths in percent
  • Responsive, linearizable, mobile-ready
  • Browser support: IE9 and up, Chrome latest, Safari latest, Firefox latest, Opera latest, major mobile browsers
  • 2 variants: labels left (for most forms) and labels above (small or narrow forms)
  • Doesn't break with multi-line labels
  • Client-side validation via HTML5 input types and JavaScript support
  • Inline error messages
  • Backend/templating friendly
    • Suitable for using label/input/error includes/snippets
    • Slim and flexible markup

Toolbelt

The file base/_toolbelt.scss contains a collection of useful helper mixins, placeholders and classes.

Global breakpoint configuration

If you are creating a responsive site, set your breakpoints first, by editing the Sass-variable $breakpoints in sass/base/_vars.scss. The benefit of working with this variable is the ability to change your media queries globally without overlooking one. Add more breakpoints as you like.

You may profit by the following mixins and classes that rely on this variable.

Example SCSS code

$breakpoints: small 500px, medium 700px, big 900px;

@mixin bp

Dynamic breakpoint mixin for responsive webdesign. Use it to create min-width media queries tied to your breakpoints.

SCSS

/*
# @mixin bp

@param {String} $breakpoint - breakpoint name
@require $breakpoints
*/
@mixin bp($breakpoint) {
    @if map-has-key($breakpoints, $breakpoint) {
        @media (min-width: map-get($breakpoints, $breakpoint)) {
            @content;
        }
    }
    @else {
        @warn "Unfortunately, no value could be retrieved from `#{$breakpoint}`. "
            + "Available breakpoints are: #{map-keys($breakpoints)}.";
    }
}

Example SCSS code

$breakpoints: (
    small:  500px,
    medium: 700px,
    big:    900px
) !default;

.wrapper {
    width: 220px;
    @include bp(small) {
        width: 460px;
    }
    @include bp(medium) {
        width: 660px;
    }
}

@mixin bp-max

Another dynamic breakpoint mixin creating max-width media queries instead.

SCSS

/*
# @mixin bp-max

@param {String} $breakpoint - breakpoint name
@require $breakpoints
*/
@mixin bp-max($breakpoint) {
    @if map-has-key($breakpoints, $breakpoint) {
        @media (max-width: map-get($breakpoints, $breakpoint)) {
            @content;
        }
    }
    @else {
        @warn "Unfortunately, no value could be retrieved from `#{$breakpoint}`. "
            + "Available breakpoints are: #{map-keys($breakpoints)}.";
    }
}

@mixin hd

"Retina" breakpoint mixin.

SCSS

/*
# @mixin hd

@param {Integer} $ratio - device pixel ratio
@param {String} $resolution - dpi-resolution
*/
@mixin hd($ratio: 2, $resolution: 192dpi) {
    @media (-min-device-pixel-ratio: $ratio), (min-resolution: $resolution) {
        @content;
    }
}

Responsive helper classes

The following classes are generated according to your breakpoint configuation.

SCSS

/*
# Dynamic classes based on $breakpoints.

Dont't show an element until or hide it starting at a specific breakpoint.

$breakpoints: (
    small:  500px,
    medium: 700px,
    big:    900px
) !default;

.reveal-small {}
.reveal-medium {}
.reveal-big {}
.hide-small {}
.hide-medium {}
.hide-big {}
*/
@each $breakpoint in $breakpoints {
    $name: nth($breakpoint, 1);
    $width: nth($breakpoint, 2);
    .hide-#{$name} {
        @media (min-width: $width) {
            display: none !important;
        }
    }
    .reveal-#{$name} {
        display: none !important;
        @media (min-width: $width) {
            display: block !important;
        }
    }
}

#responsive status

The HTML-code <span id="responsive-status"></span> has to be placed inside the main layout file. Its css-attribute font-size will be set to the name of the current active breakpoint (min-width). The reason behind this is to tell JavaScript the status of the layout of your website and to launch some JavaScript/jQuery functions according to your breakpoints.

Last but not least, <body> gets a class depending on the current breakpoint, prefixed with "responsive-". For example <body class="responsive-medium"> when the medium breakpoint is active now.

SCSS

#responsive-status {

jQuery

function checkResponsiveStatus() {
    var status = $('#responsive-status').css('font-family');

    if (status != laststatus) {

        // Set <body> classname to responsive-{name of current breakpoint}
        $('body').removeClass (function (index, css) {
            return (css.match (/(^|\s)responsive-\S+/g) || []).join(' ');
        });
        $('body').addClass("responsive-" + status);

        // Call functions tied to current breakpoint
        // Add your own cases if you like
        switch(status) {
            case "mobile":
                console.log("JavaScript-Code for mobile resolution");
                break;

            case "small":
                console.log("JavaScript-Code for small resolution");
                break;

            case "medium":
                console.log("JavaScript-Code for medium resolution");
                break;

            case "big":
                console.log("JavaScript-Code for big resolution");
                break;
        }
    }
    laststatus = status;
}

z-index Management

Gather all z-indexes into a single map, and stick to them. Then you know how many layers the application uses, making it easy to understand what’s going on (and, sooner or later, to debug).

SCSS

/*
# @function z

@param {String} $z - layer name
@require $z-layers
*/
@function z($key) {
    @if map-has-key($z-layers, $key) {
        @return map-get($z-layers, $key);
    }

    @warn "Unknown `#{$key}` in $z-layers.";
    @return null;
}

Example SCSS-Code

$z-layers: (
    header:    30,
    content:   10,
    footer:    20
);

.header {
    z-index: z(header);
}

REM font-size and line-height

With px-fallback.
http://css-tricks.com/snippets/css/less-mixin-for-rem-font-sizing/ 

SCSS

/*
# REM font-size and line-height with px-fallback

@param {Integer} $size - font-size (without px)
@param {Integer} $line - line-height (without px)
*/
@mixin font-size($sizeValue: 16, $line: $sizeValue * 1.5) {
    font-size: ($sizeValue) + px;
    line-height: ($line) + px;
    font-size: ($sizeValue / 10) + rem;
    line-height: ($line / 10) + rem;
}

Example SCSS-Code

p {
    @include font-size(16, 24);
}

Truncate Text

Ensuring that your text element doesn’t overflow its container and breaks nicely. It takes one parameter, which is any of the valid text-overflow values.
http://hackingui.com/front-end/10-best-scss-utilities/ 

SCSS

/*
# @mixin truncate-text

@param {String} $overflow - text-overflow value
*/
@mixin truncate-text( $overflow: ellipsis ) {
    overflow: hidden;
    white-space: nowrap;
    text-overflow: $overflow; // values are: clip, ellipsis, or a string
}

Caret

Creates a right angled triangle pointing to a direction specified by $direction.

Example

Me haz caret 
SCSS

/*
# @mixin caret

@param {String} $direction - top, right, bottom or left
@param {String} $size - px-value
@param {String} $color - color-value
*/
@mixin caret($direction: top, $size: 10px, $color: #333) {
    $opposite: opposite-position($direction);
    border: $size solid transparent;
    border-#{$opposite}: $size solid $color;
    border-#{$direction}: 0;
    height: 0;
    width: 0;
}

Example SCSS-Code

.caret01 {
    @include inline-block;
    position: relative;
    padding: 10px;
    background-color: $blue1;
    color: $white;
    &:before {
        position: absolute;
        bottom: -10px;
        left: 50%;
        @include translate(-50%, 0);
        content: "";
        @include caret(bottom, 10px, $blue1);
    }
}

Gradient Transition

@mixin gradient-transition creates a :hover-transition-effect with a gradient.

Example

Me haz transition 
SCSS

/*
# @mixin gradient-transition

@param {String} $start: color-value
@param {String} $end: color-value
@param {String} $duration: transition duration
*/
@mixin gradient-transition( $start, $end, $duration ){
    background-size: 100%;
    background-image: linear-gradient($start, $end);
    position: relative;
    z-index: 100;
    &:before {
        background-image: linear-gradient($end, $start);
        content: '';
        display: block;
        height: 100%;
        position: absolute;
        top: 0; left: 0;
        opacity: 0;
        width: 100%;
        z-index: -100;
        @include transition-property(opacity);
        @include transition-duration($duration);
    }
    &:hover {
        &:before {
            opacity: 1;
        }
    }
}

Example SCSS-Code

.transition1 {
    @include inline-block;
    padding: 10px;
    border: 1px solid $grey4;
    color: $text-color;
    @include gradient-transition($grey5, $grey4, 0.25s);
}
